  10. Visual Formatting, Fonts and Horizontal Rules  
  10.1 Background Color  
  This attribute sets the background color of the canvas for the document body (the body element) or for tables (the table, tr, th, and td elements). Additional attributes for specifying text color can be used with the body element.

Eg.1 This attribute is best set using style sheets as follows:

 
<style type="text/css">
{}
body {
  background-color: "#FFFFFF";
  font-family: "arial";
  font-size: "13px";
  color: "#000000"
  }
table {background-color: "#FFFFFF"}
tr {background-color: "#FFFF99"}
td {background-color: "#99FFFF"}
th {background-color: "#99FF99"}
</style>

When creating an external style sheet, always use the extra set of curly brackets immediately after the opening style tag. If the style sheet is within the document (embedded style sheet) you can omit this set of curly brackets.

  
     
  10.2 Text Alignment  
  It is possible to align block elements (tables, images, objects, paragraphs, etc.) on the canvas with the align attribute. However, we only discuss the meaning of the align attribute for text here.

align = left|center|right|justify

This attribute specifies the horizontal alignment of its element with respect to the surrounding context.

Possible values are:

  • left: text lines are rendered flush left.
  • center: text lines are centered.
  • right: text lines are rendered flush right.
  • justify: text lines are justified to both margins.

The default depends on the base text direction. For left to right text, the default is align=left, while for right to left text, the default is align=right.

Eg.2 We can align text using style sheets as follows:

 
<style type="text/css">
h2 { text-align: center}
</style>
 
<body>
<h2>This Heading Will Be Centered</h2>
</body>

Eg.3 The above style sheet would center all h2 declarations. In order to reduce the scope of the style we can set the class attribute on the element as follows:

 
<style type="text/css">
h2.centered {text-align: center}
</style>
 
<body>
<h2 class="centered">This Heading Will Be Centered</h2>
<h2>This Heading Is Not Centered</h2>
</body>

Eg.4 Similarly, you right align a paragraph with the align attribute as follows:

 
<style type="text/css">
p.rightpara {text-align: right}
</style>
 
<body>
<p class="rightpara">...Right aligned paragraph of text...</p>
<p>...Default aligned paragraph of text...</p>
</body>

Eg.5 To right align a series of paragraphs, group them with the div element:

 
<style type="text/css">
div.rightpara {text-align: right}
</style>
 
<body>
<div class="rightpara">
<p>...first paragraph of text...</p>
<p>...second paragraph of text...</p>
<p>...third paragraph of text...</p>
</div>
</body>

Eg.6 The entire document can be centered using style sheets as follows:

 
<style type="text/css">
body {text-align: center}
</style>
 
<body>
 ...the document body is centered...

</body>

  
     
  10.3 Floating Objects  
  Images and objects may appear directly "in-line" or may be floated to one side of the page, temporarily altering the margins of text that may flow on either side of the object.

Float An Object

The float property for objects, images, tables, frames, etc., causes the object to float to the left or right margin. Floating objects generally begin a new line. This property takes the following values:

  • left: Floats the object to the current left margin. Subsequent text flows along the image's right side.
  • right: Floats the object to the current right margin. Subsequent text flows along the image's left side.

Eg.7 The following example shows how to float an img element to the current left margin of the document using the float property and style sheets:

 
<style type="text/css">
img {float: "left"}
</style>
 
<body>
<img src="someimage.gif" alt="An image">
</body>

Float text around an object

The clear property can be used with the br element to control text flow around floating objects.

clear = none|left|right|both

Specifies where the next line should appear in a visual browser after the line break caused by the br element. This property takes into account floating objects (images, tables, etc.). Possible values are:

  • none: The next line will begin normally. This is the default value.
  • left: The next line will begin at nearest line below any floating objects on the left-hand margin.
  • right: The next line will begin at nearest line below any floating objects on the right-hand margin.
  • both: The next line will begin at nearest line below any floating objects on either margin.

Lets consider a situation where text flows to the right of an image until a line is broken by a br element as follows:

+++++++++ --------
|       | --------
| image | --<br>
|       |
+++++++++

If the clear property is set to none, the line following the br element will begin immediately below it at the right margin of the image as follows:

+++++++++ --------
|       | --------
| image | --<br>
|       | ------
+++++++++

If the clear property is set to left or all, the next line will appear as shown below. Using the following style sheet, you could specify that all line breaks should behave this way for objects (images, tables, etc.) floating against the left margin:

 
<style type="text/css">
br { clear: "left" }
</style>
+++++++++ --------
|       | --------
| image | --<br>
|       |
+++++++++
----------------

To specify this behavior for a specific instance of the br element, you could use the id attribute in the style information as follows:

 
<style type="text/css">
br#leftbr { clear: "left" }
</style>
+++++++++ --------
|       | --------
| image | --<br id="leftbr">
|       |
+++++++++

----------------

  
     
  10.4 Fonts  
  The use of the following HTML elements to specify font information is discouraged in favor of style sheets.

The font style elements are:

  • tt: Renders as teletype or monospaced text.
  • i: Renders as italic text style.
  • b: Renders as bold text style.
  • big: Renders text in a "large" font.
  • small: Renders text in a "small" font.
  • strike and s: Deprecated elements. Render strike-through style text.
  • u: Deprecated element. Renders underlined text.

It is possible to achieve a much richer variety of font effects using style sheets. Below are the various properties and permitted values for setting the style sheet font properties:

  1. { font-family: family-name | generic-family }
    • where font-family is the name of a font, e.g. verdana, arial, times etc. and
    • the generic-families are defined as:

"serif" (e.g. Times)

"sans-serif" (e.g. Helvetica)

"cursive" (e.g. Zapf-Chancery)

"fantasy" (e.g. Western)

"monospace" (e.g. Courier)

  1. { font-weight: normal| bold | bolder | lighter | 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900}

Internet Explorer currently only supports normal and bold.

  1. { font-size: [xx-large | x-large | large | medium | small | x-small | xx-small] | [larger | smaller] | <percentage> | <length>}
  2. { font-style: normal| italic | oblique}

Internet Explorer currently only supports italic and normal.

  1. { font-variant: normal| small-caps}

Sets a font variation of normal or small caps.

Eg.8 As an example, you can add the following code to the body element properties in the style sheet to set the style sheet font properties for the main text:
 font-family: "verdana"; font-weight: "normal"; font-size:"13px"; font-style: "normal"
The properties for the body selector (all elements are referred to as selectors when used in style sheets) should then look like this:

 
body { 
background-color: "#FFFFFF";
color: "#000000";
margin-top: "5px";
margin-bottom: "5px"; 
margin-left: "10px";
margin-right: "10px";
font-family: "verdana";
font-weight: "normal";
font-size:"13px";
font-style: "normal" 
}

To change the font properties for a particular paragraph or heading, just use the style attribute for that particular element to do so. For example, to change the font properties for a particular paragraph use the following style attribute and values:
 <p style="font-family: arial; font-weight: normal; font-size: 13px; font-style: normal">The text in this paragraph should then look like this.</p>

The hr element causes a horizontal rule to be rendered by visual user agents. The amount of vertical space left between a horizontal ruled line and the content that surrounds it depends on the user agent.
