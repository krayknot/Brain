  4. Creating an HTML Document  
  4.1 Starting-Off The Document  
  As stated on the previous page, there are certain elements that are required in an HTML document's structure. The easiest way to proceed in creating an HTML document is to type in the required elements at the start. Then add to your document by inserting elements and adding content between the starting and ending tags of the existing elements. The starting point is as it looks in the editor now (i.e. the default text in the editor).

At this point your page has a DOCTYPE Declaration right at the beginning, followed by a head and a body section inside the html tags. You also have a title element, between the head elements, where you should remove "Untitled" and replace it with your chosen title. Every HTML document must have a title element in the head section.

The title of the document appears in the titlebar of the user's browser and if the user Bookmarks the page or saves it as a Favorite, then it is the title that is added as the description of the link. The title should be as descriptive as possible because this is what many search engines use to index your page.

  
     
  4.2 Specifying meta Tag Data  
 

Syntax    <meta>

List of meta Attributes

The meta element can be used to identify properties of a document (e.g., author, expiration date, content, keywords, etc.) and assign values to those properties. Meta data is placed between the title tags and the closing head tag.

Each meta element specifies a property/value pair. The name attribute identifies the property and the content attribute specifies the property's value.

As an example, the following declaration sets a value for the Author property:

<meta name="Author" content="Author's Name Here">

Below are some example meta tags that you can use. Just change the data in the "content" section where appropriate. The first two meta tags (keywords and description) are the most important as far as search engine rankings are concerned.

<meta name="keywords" content="computer hardware, monitors, cpu's, dvd's, scanners,
printers, color printers, laser printers, laser color printers, inkjet printers,
inkjet color printers, inkjet cartridges, toners"> 
(The above tag is used by most search engines to index your site. 
Try to think of as many keywords that people may associate with your site.)
 
<meta name="description" content="Listings of all the leading brands of computer 
hardware and accessories. Full technical details and photos on all equipment 
available and discounted prices on most items. All listed inventory is immediately 
available from our warehouse. Secure shopping cart; ordered items shipped the next day">
(The above tag is used by most search engines as the description of your site. It is also 
used by many Internet programs that query search engines to give a description of each 
of the sites listed on the search results page.)
 
<meta name="rating" content="General">
(The above tag is used to give an indication of the type of audience
for which the site is suitable. "General" is the widest category.)
 
<meta name="revisit-after" content="30 days">
(The above tag gives the period after which the search robot should
re-visit the site to see if there are any changes (expressed in days). Generally the
time interval stated here is 30 days. Don't make it shorter without good reason.
 
<meta http-equiv="creation_date" content="Tues, May 22, 2001 14:30:00 GMT">
(The above tag states when the document was created.)
 
<meta http-equiv="expires" content="Sun, 27 May, 2001 14:30:00 GMT">
(The above tag states when the document will cease to be valid.
Some search engines will remove the document after this date.)
 
<meta http-equiv="copyright" content="Your name OR Company name - 2001">
(The above tag gives copyright information.)
 
<meta name="robots" content="NOINDEX,NOFOLLOW">
(The above tag instructs search engine robots not to include this
page in their index (effected by the word NOINDEX) and not to follow the links on this 
page (effected by the word NOFOLLOW). You can remove one of the words to suit your needs.)
 
<meta http-equiv="window-target" content="_top">
(The above tag gives the name of the window in which the page is to
be loaded. Specifying "_top" prevents the page from loading in someone else's frames.)
 
<meta http-equiv="pragma" content="no-cache">
(The above tag prevents browsers from cacheing the page.)

PLACEMENT OF THE META TAGS IN AN HTML DOCUMENT

<html>
<head>
        <title>Computer hardware and accessories</title>
<meta name="keywords" content="computer hardware, monitors, cpu's, dvd's, scanners,
printers, color printers, laser printers, laser color printers, inkjet printers,
inkjet color printers, inkjet cartridges, toners"> 
<meta name="description" content="Listings of all the leading brands of computer 
hardware and accessories. Full technical details and photos on all equipment 
available and discounted prices on most items. All listed inventory is immediately 
available from our warehouse. Secure shopping cart; ordered items shipped the next day"> 
<meta name="rating" content="General">
<meta name="revisit-after" content="30 days">
<meta http-equiv="creation_date" content="Tues, May 22, 2001 14:30:00 GMT">
<meta http-equiv="expires" content="Sun, 27 May, 2001 14:30:00 GMT">
<meta http-equiv="copyright" content="Your name OR Company name - 2001">
<meta name="robots" content="NOINDEX,NOFOLLOW">
<meta http-equiv="window-target" content="_top">
<meta http-equiv="pragma" content="no-cache">
</head>
<body>
  ...document's content goes here...
</body>
</html>
 
     
  4.3 The Document body  
 

Syntax    <body>...</body>

List of body Attributes

The document properties are set by using the attributes of the body element. The document's background image (the value of this attribute is a URI that designates an image resource) or background color, it's text color, the colors of the different states of the links and the page margins are set using this element's attributes. All colors on an HTML document should be represented as hexadecimal code, e.g. red=#FF0000, blue=#0000FF and green=#00FF00. You should always include the hash sign (#) before the hexadecimal code.

Eg.1 The following HTML example sets the background color of the document to white (#FFFFFF), the text foreground color to black (#000000), and the color of hyperlinks to blue (#0000FF) initially, yellow (#FFFF00) when selected (activated), and purple (#800080) once visited. However, these attributes have been deprecated in favor of style sheets (Eg.2 below).

Deprecated Example

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
<html>
<head>
 <title>Example document properties</title>
</head>
<body bgcolor="#FFFFFF" text="#000000" link="#0000FF" alink="#FFFF00" vlink="#800080"
topmargin="5" bottommargin="5" leftmargin="10" rightmargin="10" marginwidth="10" 
marginheight="5">
  ... document's content goes here...
</body>
</html>

The page margins can also be set in the style sheet using the appropriate properties.

Example margin declarations are as follows:

body { 
  margin-top: "5px"; 
  margin-bottom: "5px"; 
  margin-left: "10px";             /* top and bottom page margins are 5px, 
  margin-right: "10px"                right and left page margins are 10px.     
  }                                   This example produces the same 
                                      results as the next one */
  
body { margin: 5px 10px 5px 10px } /* top and bottom margins are 5px,
                                      right and left margins are 10px */
                                                                       
div { margin: 5px 10px }           /* top and bottom margins are 5px,
                                      right and left margins are 10px */
                                                                       
p { margin: 10px 20px }            /* top and bottom margins are 10px,
                                      right and left margins are 20px */
                                                                       
p { margin: 10px }                 /* all the margins are 10px */

With regard to examples 2 to 5, if four values are given, they are applied to the margins in the following order: top, right, bottom, and left respectively, i.e. starting on TOP and going in a CLOCKWISE direction. If one value is given, it applies to ALL sides. If two or three values are given, the missing value(s) is/are taken from the opposite side.

You can use any of the following most widely used length units:

  • px (pixels; relative to the resolution of the viewing device)
  • pt (points; 1pt=1/72in)
  • em (ems; the 'font-size' of the relevant font)
  • ex (x-height; the 'x-height' of the relevant font)

Eg.3 Using external (linked) style sheets gives you the flexibility to change the presentation without revising the source HTML document. For example, you can save the style sheet (everything between, and including, the <style type="text/css"> and </style> tags in eg.2 above as nameofstyle.css. The resulting external style sheet is as follows (notice the added extra set of curly brackets immediately after the opening style tag):
 
 <style type="text/css">
 {}
  body { 
  background-color: "#FFFFFF"; 
  color: "#000000"; 
  margin-top: "5px"; 
  margin-bottom: "5px"; 
  margin-left: "10px"; 
  margin-right: "10px"
  }

  a:active { color: "#FFFF00" }

Notice that the external style sheet does not contain any DTD, <html>, <head>, <title> or <body> tags as in a normal web page. It is saved as nameofstyle.css just as you see it. You can add and save other selectors, their properties and values to the style sheet as you go along (see section 3.5 below). In CSS a selector is synonymous with an HTML element and a property is synonymous with an HTML attribute. In the above style sheet, body is a selector, background-color is a property and #FFFFFF is the property value. Likewise a:link = selector, color = property and #0000FF = property value.

Then just link to the external style sheet from each web page as shown in the example below.

Just place the link tag with all of it's attributes and attribute values into each HTML document in the same position as in the example below. The file "nameofstyle.css" (change nameofstyle to whatever name you want and then save the file with a .css extension) must be saved in the same folder as your web pages.

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
<html>
<head>
 <title>Example document properties</title>
 <link rel="stylesheet" type="text/css" href="nameofstyle.css">
</head>
<body>
  ...document's content goes here...
</body>
</html>
 
     
  4.4 Using An Image As The Background  
  To use an image for the page background instead of a color, in the stylesheet in Eg.2 above, make the following change to the body element properties - change background-color: "#FFFFFF"; to
 background-image:url("image_name.gif"); [Don,t leave out the semi-colon]. If the image is not in the same directory as your web pages then you must state the relative path to the image, e.g. if the image is in a directory named images then the path would be: background-image:url("images/image_name.gif");
However, if the image is on another server then you must state the absolute path to the image, e.g. background-image:url("http://www.sitename.com/images/image_name.gif");

If you don't want the image to be tiled so that it covers the whole page, add the following code to the same line of code: background-repeat:"no-repeat"; In this case you should leave the background-color: "#FFFFFF"; code in place and not remove it. Replace #FFFFFF with the hexadecimal code of whatever color you are going to use as the page background. The same applies when you use
 background-repeat: "repeat-x" or
 background-repeat: "repeat-y" (See the next paragraph).

If you want the image to be tiled only along the x-axis, replace "no-repeat" in the above code with "repeat-x"
If you want the image to be tiled only along the y-axis, then replace "no-repeat" with "repeat-y"

If you want the background image to be static, i.e. non-scrolling, then add the following code to the same line of code: background-attachment:"fixed";

Note that the background-repeat and background-attachment properties only function in Internet Explorer.

 
     
  4.5 Setting The Style Sheet Font Properties  
  Below are the various properties and permitted values for setting the style sheet font properties:
  1. { font-family: family-name | generic-family }
    • where family-name is the name of a font, e.g. verdana, arial, times etc. and
    • the generic-families are defined as:

"serif" (e.g. Times)

"sans-serif" (e.g. Helvetica)

"cursive" (e.g. Zapf-Chancery)

"fantasy" (e.g. Western)

"monospace" (e.g. Courier)

  1. { font-weight: normal| bold | bolder | lighter | 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900}

Internet Explorer currently only supports normal and bold.

  1. { font-size: [xx-large | x-large | large | medium | small | x-small | xx-small] | [larger | smaller] | <percentage> | <length>}
  2. { font-style: normal| italic | oblique}

Internet Explorer currently only supports italic and normal.

  1. { font-variant: normal| small-caps}

Sets a font variation of normal or small caps.

As an example, you can add the following code to the body element properties in the style sheet to set the style sheet font properties for the main text:
 font-family: "verdana"; font-weight: "normal"; font-size:"13px"; font-style: "normal"
The properties for the body element should then look like this:

 
body { 
background-color: "#FFFFFF";
color: "#000000";
margin-top: "5px";
margin-bottom: "5px"; 
margin-left: "10px";
margin-right: "10px";
font-family: "verdana";
font-weight: "normal";
font-size:"13px";
font-style: "normal" 
}

The following way of setting the font properties for the main text which appears between the opening and closing body tags is still valid but has been deprecated in favor of style sheets as illustrated above:

<p><font face="verdana" color="#000000" size="2">This code produces the same text characteristics as that produced by the body properties in the style sheet above, but as already stated, has been deprecated in favor of style sheets.</font></p>

It is therefore strongly recommended that you use style sheets to control presentation and get into the practice of using style sheets from the beginning.

To change the font properties (to something that is different to that declared in the style sheet) for a particular paragraph or heading, just use the style attribute for that particular element to do so. For example, to change the font properties for a particular paragraph you can use the following style attribute and values:
 <p style="font-family: courier; font-weight: bold; font-size: 13px; font-style: normal" >The text in this paragraph should then look like this.</p>

To make text bold you have to add the text that you want to appear in bold between these opening and closing tags: <strong></strong>
To make the text italic you have to add the text that you want to appear in italic between these opening and closing tags: <i></i>

You can use multiple tags but they must be nested. For example:

<p><strong><i>...Here is the text between the tags...</i></strong></p> This is correct as the tags are nested. You can see that the first opening tag, <p>, becomes the last closing tag, </p>, and that the last opening tag, <i>, becomes the first closing tag, </i>
The opening tags can be ordered in any way but the closing tags must be closed according to a sequence in which they were opened, as explained above. An easy way to do this is to read the sequence of the tags that you have opened from right to left (i.e. read the sequence from the last opening tag to the first opening tag) and then close each tag in that sequence.

The body element is replaced by the frameset element in documents that contain framesets. This is covered in the section on frames.

  4.6 Element identifiers: the id and class attributes  
  id:- This attribute assigns a name to an element. This name must be unique in a document.

The id attribute assigns a unique identifier to an element. For example, the following headings are distinguished by their id values:

<h2 id="heading1">This is heading 1.</h2>
<h2 id="heading2">This is heading 2.</h2>

The id attribute has several roles in HTML:

  • As a style sheet selector.
  • As a target anchor for hypertext links. (Not all browsers support the id anchor at the moment and hence using both the id and name attributes is a better option for now, e.g.,
         <h2 id="heading1" name="heading1">This is heading 1.</h2>).
  • As a means to reference a particular element from a script.
  • As the name of a declared object element.
  • For general purpose processing by user agents.

class:- This attribute assigns a class name or set of class names to an element. Any number of elements may be assigned the same class name or names.

The class attribute, on the other hand, assigns one or more class names to an element; the element may be said to belong to these classes. A class name may be shared by several element instances. The class attribute has several roles in HTML:

  • As a style sheet selector (to assign style information to a set of elements).
  • For general purpose processing by user agents.

In the following example, the span element is used in conjunction with the id, name and class attributes to markup a document's messages:

 <p><span id="msg1" name="msg1" class="product">Message 1 here</span></p>
 <p><span id="msg2" name="msg2" class="info">Message 2 here</span></p>
 <p><span id="msg3" name="msg3" class="specials">Message 3 here</span></p>
 The following CSS style rules would need to be added to 
    the style sheet in order for the messages to be displayed in different 
    colors:
 
span.product { color: "#0000FF" }
span.info { color: "#00CE00" }
span.specials { color: "#FF0000" }

where span = the selector (selector in CSS and element in HTML) and
product, info and specials = the classes.

Almost all HTML elements may be assigned identifier and class information.

  4.7 Block-level and inline elements  
  Certain HTML elements that may appear within the body tags are said to be "block-level" while others are "inline", also known as "text level".

The div element defines a generic block-level container, which allows you to provide style or language information to blocks of content. The element may contain any inline or block-level element, including another div element.

span is a text-level equivalent of the div element for use within paragraphs and inline elements.

  
     
  4.8 The div and span elements  
 

Syntax    <div>...</div>

List of div Attributes

Syntax    <span>...</span>

List of span Attributes

The div and span elements, in conjunction with the id and class attributes, offer a mechanism for adding structure to documents. These elements define content to be inline (span) or block-level (div).

  4.9 Headings: The h1, h2, h3, h4, h5, h6 elements  
 

Syntax    <hx>...</hx>

where x = any number 1 to 6.

List of heading Attributes

A heading element briefly describes the topic of the section it introduces. There are six levels of headings in HTML with h1 as the most important and h6 as the least. h1 displays as the larger font size and the font sizes get progressively smaller from h1 to h6.

Eg.7 

<h1>This is a level 1 heading</h1>
<p>...paragraph of text here...</p>
<h2>This is a level 2 heading</h2>
<p>...paragraph of text here...</p>
<h3>This is a level 3 heading</h3>
<p>...paragraph of text here...</p>
<h4>This is a level 4 heading</h4>
<p>...paragraph of text here...</p>
<h5>This is a level 5 heading</h5>
<p>...paragraph of text here...</p>
<h6>This is a level 6 heading</h6>
<p>...paragraph of text here...</p>

  4.10 The address element  
 

Syntax    <address>...</address>

List of address Attributes

The address element provides contact information for a document or part of a document. Information provided by address may include the names of the people who maintain the document, links to their Web pages, e-mail addresses etc.

The following example, which would most appropriately be placed at the end of a document, gives contact information about the person maintaining the document:

 
<address>This page is maintained by <a href="../JDoe/">John Doe</a>, E-mail:
<a href="mailto:jdoe@adomain.com">jdoe@adomain.com</a></address>

This would be rendered as follows:

This page is maintained by John Doe, E-mail: jdoe@adomain.com
