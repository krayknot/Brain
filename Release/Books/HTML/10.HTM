  11. Frames  
  11.1 Introduction  
  Frames are used to divide the browser window into several parts or views. Each frame consists of an individual HTML document. Most frames pages that you see on the Internet are those where the navigation links are in the frame on the left side of the browser window. When the links in this frame are clicked, the requested web page appears in the frame on the right side. The navigation links in the frame on the left side of the browser window stay visible all the time.

Eg.1 Below is a frameset document of the type described above:

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Frameset//EN">
<html>
<head>
<title>A frameset document</title>
</head>
<frameset cols="20%, 80%" frameborder="1">
      <frame src="navigation.htm" name="navigation" scrolling="auto">
      <frame src="sitecontent.htm" name="contents" scrolling="yes">
 </frameset>
  <noframes>
      <p>This frameset document contains:</p>
      <ul>
         <li><a href="navigation.htm">Navigation Menu</a></li>
         <li><a href="sitecontent.htm">Main Contents</a></li>
      </ul>
  </noframes>
</html>

The frame layout would be as illustrated below:

  -----------------------------------------------
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |  FRAME 1   |             FRAME 2              |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
 |            |                                  |
  ----------------------------------------------- 

If the browser is unable to display frames or is configured not to, it will render the contents of the noframes element.

  
     
  11.2 Layout Of Frames  
  An HTML document that describes a frames layout (called a frameset document) has a different makeup to an HTML document without frames. A standard document has one head section and one body section. A frameset document has a head section, and a frameset section in place of the body section.

The frameset section of a document specifies the layout of views in the main browser window. In addition, the frameset section can contain a noframes element to provide alternate content for browsers that do not support frames or are configured not to display frames.

Elements that might normally be placed in the body element must not appear before the first frameset element or the frameset will be ignored.

  
     
  11.3 The frameset Element  
 

Syntax    <frameset>...</frameset>

List of frameset Attributes

The frameset element specifies the layout of the main user window in terms of rectangular subspaces.

Rows and columns

Setting the rows attribute defines the number of horizontal subspaces in a frameset. Setting the cols attribute defines the number of vertical subspaces. Both attributes may be set simultaneously to create a grid.

If the rows attribute is not set, each column extends the entire length of the page. If the cols attribute is not set, each row extends the entire width of the page. If neither attribute is set, the frame takes up exactly the size of the page.

Frames are created left-to-right for columns and top-to-bottom for rows. When both attributes are specified, views are created left-to-right in the top row, left-to-right in the second row, etc.

Eg.2 In the example below, the screen is divided horizontally in two, creating a top half and a bottom half.

 
<frameset rows="50%, 50%">
...the rest of the definition goes between these tags...
</frameset>

Eg.3 The next example creates three columns. The second column has a fixed width of 250 pixels. The first column receives 25% of the remaining space and the third column 75% of the remaining space.

 
<frameset cols="1*,250,3*">
...the rest of the definition goes between these tags...
</frameset>

Eg.4 The next example creates a 2x3 grid of subspaces.

 
<frameset rows="30%,70%" cols="33%,34%,33%">
...the rest of the definition goes between these tags...
</frameset>

Eg.5 In the next example, the first view is allotted 30% of the total browser window height. The second view is specified to be exactly 350 pixels high. The fourth frame's height is specified as "2*", so it is twice as high as the third frame, whose height is only "*" (equivalent to 1*).

 
<frameset rows="30%,350,*,2*">
...the rest of the definition goes between these tags...
</frameset>

Nested frame sets

Framesets may be nested to any level.

Eg.6 In the following example, the outer frameset divides the available space into three equal columns. The inner frameset then divides the second area into two rows of unequal height.

 
<frameset cols="33%, 33%, 34%">
     ...contents of first frame here...
     <frameset rows="40%, 60%">
        ...contents of second frame, first row here...
        ...contents of second frame, second row here...
     </frameset>
     ...contents of third frame here...

</frameset>

  
     
  11.4 The frame Element  
 

Syntax    <frame>

List of frame Attributes

The frame element defines the contents and appearance of a single frame.

Specifying the initial contents of a frame

The src attribute specifies the initial document the frame will contain.

Eg.7 The following example:

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Frameset//EN">
<html>
<head>
<title>A frameset document</title>
</head>
<frameset cols="20%, 80%" frameborder="1">
  <frameset rows="70%,30%">
      <frame src="navigation.htm" name="navigation" scrolling="auto">
      <frame src="image.gif" name="image" scrolling="no">
 </frameset>
 <frame src="sitecontent.htm" name="contents" scrolling="yes">
</frameset>
</html>

will create a layout as follows:

  ---------------------------------------------
 |            |                                |
 |            |                                |
 |            |                                |
 |            |                                |
 |   FRAME 1  |                                |
 |(NAVIGATION)|                                |
 |            |                                |
 |            |             FRAME 3            |
 |            |           (CONTENTS)           |
 |            |                                |
 |------------|                                |
 |            |                                |
 |            |                                |
 |  FRAME 2   |                                |
 |  (IMAGE)   |                                |
 |            |                                |
  ---------------------------------------------

Visual rendering of a frames window

In Eg.8 below, using the frame element and its attributes, we can specify that Frame 1 will allow scroll bars only when needed (using the auto attribute). Frame 2 will leave white space around its content (i.e. the image file, using the marginwidth and marginheight attributes) and the frame will not be resizeable (using the noresize attribute). No border will be drawn between Frames 1 and 2 (using the frameborder="0" attribute and value). A border will be drawn (by default) separating Frames 1 and 2 from Frame 3.

See List of frame Attributes

Eg.8 

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Frameset//EN">
<html>
<head>
<title>A frameset document</title>
</head>
<frameset cols="20%, 80%">
  <frameset rows="70%,30%">
      <frame src="navigation.htm" name="navigation" scrolling="auto" frameborder="0">
      <frame src="image.gif" name="image" scrolling="no" 
          marginwidth="10" marginheight="10" frameborder="0">
 </frameset>
 <frame src="sitecontent.htm" name="contents" scrolling="yes" noresize>
</frameset>

</html>

  
     
  11.5 Specifying Target Frame Information  
  target = frame-target: This attribute specifies the name of a frame where a document is to be opened.

See name in List of frame Attributes

By assigning a name to a frame using the name attribute, it can be referred to as the "target" of links defined by other elements. The target attribute may be set for elements that create links (a, link), image maps (area), and forms (form).

Eg.9 The example below illustrates how targets allow another web page to be loaded into a particular named frame. First we define the frameset document as shown below:

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
<html>
<head>
<title>A frameset document</title>
</head>
<frameset cols="20%, 80%" frameborder="1">
      <frame src="navigation.htm" name="navigation" scrolling="auto">
      <frame src="sitecontent.htm" name="contents" scrolling="yes">
 </frameset>
</html>

Eg.10 Then, in the document, sitecontent.htm, we have links which contain the target attribute whose value is "contents". In this way we establish links to the frame named "contents".

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Frameset//EN">
<html>
<head>
<title>Document containing anchors with specific targets</title>
</head>
<body>
...beginning of document sitecontent.htm...
<p>For product news 
    <a href="productnews.htm" target="contents">click here</a>.</p>
...more of document...
<p>For latest product news
    <a href="latestnews.htm" target="contents">click here</a>.</p>
</body>
</html>

Activating either link opens a new document in the frame named "contents" while the other frame named "navigation", maintains its initial contents.

When coding the links in the document "navigation.htm", the target="contents" attribute and value will have to be stated as in the above example so that the linked pages load into the frame named "contents".

  
     
  11.6 Setting The Default Target For Links  
  When many links in the same document designate the same target, as for example in the navigation page or menu, it is possible to specify the target once and dispense with the target attribute of each element. This is done by setting the target attribute of the base element.

Syntax    <base>

List of base Attributes

Let's return to the previous example, this time adding the target information by defining it in the base element and removing it from the a elements.

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Frameset//EN">
<html>
<head>
<title>Document using base element and a specific target</title>
<base href="http://www.somesite.com/" target="contents">
</head>
<body>
...beginning of document sitecontent.htm...
<p>For product news 
    <a href="productnews.htm">click here</a>.</p>
...more of document...
<p>For latest product news
    <a href="latestnews.htm">click here</a>.</p>
</body>

</html>

 

  
     
  11.7 Target Values  
  The following target values are permitted in order to specify the name of a frame or window so it may be targeted from links in other documents. Either a window-name or frame-name may be specified (#1), or one of the other special values (#2 to #5) below:
  1. window/frame name: The name of a window/frame.
  2. _blank: Specifies to load the link into a newly opened blank window. This window is not named.
  3. _parent: Specifies to load the link into the immediate parent of the document the link is in.
  4. _self: Specifies to load the link into the same window the link was clicked in.
  5. _top: Specifies to load the link into the full body of the window.

If there is no frame or window that matches the specified target, a new window is opened for the link.

  
     
  11.8 Alternate Content  
  Alternate content should be provided for those user agents that do not support frames or are configured not to display frames.

Syntax    <noframes>...</noframes>

List of noframes Attributes

The noframes element specifies content that should be displayed only by user agents that do not support frames or are configured not to display frames. The contents of a noframes declaration must only be displayed by user agents that support frames when they are configured not to display frames. However, user agents that do not support frames must always display the contents of a noframes declaration.

The noframes element may be used at the end of the frameset section of a document that uses the frameset DTD.

Eg.11 

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Frameset//EN">
<html>
<head>
<title>A frameset document</title>
</head>
<frameset cols="20%, 80%" frameborder="1">
      <frame src="navigation.htm" name="navigation" scrolling="auto">
      <frame src="sitecontent.htm" name="contents" scrolling="yes">
 </frameset>
  <noframes>
      <p>For the No Frames version <a href="noframes.htm">click here</a>.</p>
  </noframes>

</html>

 

  
     
  11.9 Inline frames: the iframe element  
 

Syntax    <iframe>...</iframe>

List of iframe Attributes

To insert a frame within a block of text the iframe element can be used.

The information to be inserted inline is designated by the src attribute of this element. The contents of the iframe element should only be displayed by user agents that do not support frames or are configured not to display frames.

Eg.12 In the following example, user agents that support frames will place an inline frame surrounded by a border in the middle of the text.

 
<iframe src="somedoc.htm" width="300" height="400"
           scrolling="auto" frameborder="1">
[Your user agent is unable to display frames. However, 
to visit the document
<a href="somedoc.htm">click here.</a>]
</iframe>

Inline frames may not be resized and therefore do not take the noresize attribute.
