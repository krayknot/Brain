  6. Unordered, Ordered and Definition Lists  
  6.1 Introduction  
  There are three types of lists:
  • Unordered Lists (ul).
  • Ordered Lists (ol).
  • Definition Lists (dl).

Eg 1. An unordered list is created using the ul element and contains unordered information. The following is an example of how the above list was created:

 
<ul>
<li>Unordered Lists (ul).</li>
<li>Ordered Lists (ol).</li>
<li>Definition Lists (dl).</li>
</ul>

Eg 2. An ordered list is created using the ol element and should contain information where order is important, e.g. as in a recipe:

 
<ol>
<li>Weigh 200g of flour.</li>
<li>Add 100ml of milk.</li>
<li>Add 50g of sugar.</li>
<li>Mix thoroughly.</li>
<li>Bake in oven for 30 minutes at 250 degrees centigrade.</li> 
</ol>

It is rendered as follows:

  1. Weigh 200g of flour.
  2. Add 100ml of milk.
  3. Add 50g of sugar.
  4. Mix thoroughly.
  5. Bake in oven for 30 minutes at 250 degrees centigrade.

Eg 3. A definition list is created using the dl element and usually consist of a series of term/definition pairs:

 
<dl>
<dt>Hotspot</dt>
<dd>A clickable area of text or graphics.</dd>
<dt>Menu</dt>
<dd>A list or group of choices available in a given computing situation.</dd>
<dt>Palette</dt>
<dd>A collection of available colors.</dd>
</dl>

It is rendered as follows:

Hotspot

A clickable area of text or graphics.

Menu

A list or group of choices available in a given computing situation.

Palette

A collection of available colors.

Eg 4. Lists may also be nested and different list types may be used together. In the following example, a definition list contains an unordered list (the ingredients) and an ordered list (the procedure):

 
<dl>
<dt><strong>Ingredients:</strong></dt>
<ul>
<li>Flour</li>
<li>Milk</li>
<li>Sugar</li>
</ul>
<dt><strong>Procedure:</strong></dt>
<ol>
<li>Weigh 200g of flour.</li>
<li>Add 100ml of milk.</li>
<li>Add 50g of sugar.</li>
<li>Mix thoroughly.</li>
<li>Bake in oven for 30 minutes at 250 degrees centigrade.</li>
</ol>
</dl>

It is rendered as follows:

Ingredients:

  • Flour
  • Milk
  • Sugar

Procedure:

  1. Weigh 200g of flour.
  2. Add 100ml of milk.
  3. Add 50g of sugar.
  4. Mix thoroughly.

Bake in oven for 30 minutes at 250 degrees centigrade.

Syntax    <li>...</li>

List of li Attributes

Ordered and unordered lists are rendered in an identical manner except that visual user agents number ordered list items. User agents may present those numbers in a variety of ways as explained later. Unordered list items are not numbered and may also be presented in a number of ways as illustrated later.

Both types of lists are made up of sequences of list items defined by the li element.

The basic structure of an unordered list is as follows:

 
<ul>
<li>First list item</li>
<li>Second list item</li>
</ul>

It is rendered as a bulleted list as follows:

  • First list item
  • Second list item

To make the above list an ordered list, just change the <ul> tag to <ol> and the </ul> tag to </ol> as follows:

 
<ol>
<li>First list item</li>
<li>Second list item</li>
</ol>

It is then rendered as a numbered list as follows:

  1. First list item
  2. Second list item

In ordered lists, it is possible to continue list numbering from a previous list by setting the list item (li) value attribute. Numbering continues from the new value for subsequent list items. For example:

 
<ol>
<li value="25">makes this list item number 25.</li>
<li>makes this list item number 26.</li>
<li value="30">makes this list item number 30.</li>
<li>makes this list item number 31.</li>
<li>makes this list item number 32.</li>
</ol>

It is rendered as follows:

  1. makes this list item number 25.
  2. makes this list item number 26.
  1. makes this list item number 30.
  2. makes this list item number 31.

makes this list item number 32.

  
     
  6.3 Definition lists: the dl, dt, and dd elements

List of dt Attributes

Syntax    <dd>...</dd>

List of dd Attributes

Definition lists vary only slightly from other types of lists in that list items consist of two parts: a definition term and a definition description. The definition term is given by the dt element and is restricted to inline content. The definition description is given with a dd element that contains block-level content.

Here's an example that was given to you earlier:

 
<dl>
<dt><strong>Hotspot</strong></dt>
<dd>A clickable area of text or graphics.</dd>
<dt><strong>Menu</strong></dt>
<dd>A list or group of choices available in a given computing situation.</dd>
<dt><strong>Palette</strong></dt>
<dd>A collection of available colors.</dd>
</dl>

It is rendered as follows:

Hotspot

A clickable area of text or graphics.

Menu

A list or group of choices available in a given computing situation.

Palette

A collection of available colors.

In this case the definition terms are rendered in bold type. Look at the code above to see how it was done using the strong element.

  
     
  6.4 Visual rendering of lists  
  For both ol and ul, the type attribute specifies rendering options for visual user agents.

For the ul element, possible values for the type attribute are disc, square, and circle. The default value depends on the level of nesting of the current list.

A graphical user agent might render this as:

  • for the value "disc"
  • for the value "circle"
  • for the value "square"

For the ol element, possible values for the type attribute are shown in the table below. These values are
 case-sensitive:

type

list-style-type

NUMBERING STYLE

1

decimal

1, 2, 3, ...

-

decimal-leading-zero

01, 02, 03, ..., 10, 11,...

a

lower-alpha

a, b, c, ...

A

upper-alpha

A, B, C, ...

i

lower-roman

i, ii, iii, ...

I

upper-roman

I, II, III, ...

Note that the type attribute is deprecated and the list-style-type should be handled through style sheets.
Also note that the decimal-leading-zero value is not supported by all browsers at the moment. However, the list should still be rendered as though the decimal value was used.

Eg 5. For example, using CSS, one may specify that the numbering style for list elements in a numbered list should be uppercase roman numerals. In the example below, every ol element belonging to the class "mystyle" will have uppercase roman numerals in front of its list items.

 
<head>
<title>List Style Type</title>
<style type="text/css">
ol.mystyle { list-style-type: upper-roman }
</style>
</head>
<body>
<ol class="mystyle">
<li>First list item</li>  
<li>Second list item</li>
<li>Third list item</li>
<li>Fourth list item</li>
</ol>
</body>

It would be rendered as follows:

  1. First list item
  2. Second list item
  3. Third list item
  4. Fourth list item
