  2.1 What Is The World Wide Web?  
  The World Wide Web (a.k.a. the Internet or Web) is a Worldwide network of inter-connected computers that use the TCP/IP network protocols to facilitate data transmission and exchange of information resources between these computers. To do this, the Web relies on three mechanisms to make these resources readily available to the widest possible audience:
  1. A uniform naming scheme for locating resources on the Web (e.g., URIs).
  2. Protocols, for access to named resources over the Web (e.g., HTTP).
  3. Hypertext, for easy navigation among resources (e.g., HTML).
 
     
  2.2 Introduction To URIs  
  Every resource available on the Internet, i.e. an HTML document, image, program, etc., has an address that may be encoded by a Universal Resource Identifier, or "URI".

URIs typically consist of three pieces:

  1. The naming scheme of the mechanism used to access the resource.
  2. The name of the machine hosting the resource.
  3. The name of the resource itself, given as a path.

Consider the following URI that designates yoursite's products page:

http://www.yoursite.com/products.html

This URI may be read as follows: There is a document available via the http protocol, residing on the machine www.yoursite.com, accessible via the path /products.html. Other schemes you may see in HTML documents include "mailto" for e-mail and "ftp" for FTP (file transfer protocol).

The following example of a URI refers to a user's e-mail box:

<a href="mailto:yourname@yoursite.com">Description of Link Here e.g. Your Name</a>.

Note: Most readers will probably be familiar with the term URL (Universal Resource Locator) and not the term URI. URLs form a subset of the more general URI naming scheme.

  
     
  2.3 Fragment Identifier's  
  Some URIs refer to a location within a resource. This kind of URI ends with "#" followed by an anchor identifier (called the fragment identifier). For instance, here is a URI pointing to an anchor named specials:

http://www.yoursite.com/products.html#specials

Clicking on such a link will take the user to the specified page which will then automatically scroll to the fragment identifier.

  2.4 Relative URIs  
  A relative URI doesn't contain any naming scheme information. Its path generally refers to a resource on the same machine as the current document. Relative URIs may contain relative path components (e.g., "../" means one level up in the hierarchy defined by the path), and may contain fragment identifiers.

Relative URIs are resolved to full URIs using a base URI. As an example of relative URI resolution, assume we have the base URI "http://www.yoursite.com/index.html". The relative URI in the following markup for a hypertext link:

<a href="products.html">Products</a>

would expand to the full URI "http://www.yoursite.com/products.html", while the relative URI in the following markup for an image

<img src="../images/image_name.gif" alt="My Picture">

would expand to the full URI "http://www.yoursite.com/images/image_name.gif".

  
     
  2.5 What is HTML?  
  HTML is an acronym for HyperText Markup Language and is the publishing language used on the World Wide Web. It is a subset of an international standard for electronic document exchange called SGML (Standard Generalized Markup Language) and a language that all computers with the necessary user agents installed on them can understand. User agents include visual browsers (text-only and graphical), non-visual browsers (audio, Braille), search robots, proxies, etc.

HTML gives you the means to:

  • Publish online documents with text, tables, photos, etc.
  • Retrieve online information by clicking on hypertext links.
  • Design forms for conducting transactions with remote services, for use in searching for information, ordering products, etc.
  • Include video clips, sound clips, and other applications directly into their documents.

HTML 4.0 extends HTML with mechanisms for style sheets, scripting, frames, embedding objects, improved support for right to left and mixed direction text, richer tables, and enhancements to forms

  
     
  2.6 Compound Documents  
  HTML offer a standard mechanism for embedding generic media objects and applications in HTML documents. The object element (together with its more specific ancestor elements img and applet) provides a mechanism for including images, video, sound, mathematics, specialized applications, and other objects in a document. (The object element is intended to replace the more specific img and applet elements, as well as the proprietary embed and bgsound elements, though a lack of browser support and bugs in supporting browsers make the other elements a better choice for now).  
     
  2.7 Style Sheets  
  Style sheets simplify HTML markup and allow greater influence over presentation. They give you control over the presentation of documents, e.g. font information, alignment, colors, etc. Style information can be specified for individual elements or groups of elements and may be specified in an HTML document (embedded style sheet) or in external style sheets. (Discussed in the section on Style Sheets)  
     
  2.8 Scripting  
  Through scripts, you can create dynamic Web pages and use HTML as a means to build networked applications. (Discussed in the section on Scripts)
