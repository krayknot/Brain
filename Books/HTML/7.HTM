  8. Links and Anchors  
  8.1 Introduction  
  A link is a connection from one Web resource to another. It has two ends, called anchors and a direction. The link starts at the "source" anchor and points to the "destination" anchor, which may be any resource on the Web (e.g., an HTML document, an element within the same HTML document that contains the link, a program, an image, a video file, a music/sound file, etc.).  
     
  8.2 Visiting a linked resource  
  Activating a link (e.g. by clicking on it) causes the retrieval of another Web resource.

The following example contains two links, one whose destination anchor is an HTML document named "product.html" and the other whose destination anchor is a GIF image in the file "productimage.gif":

 
<body>
<p>For more product information go to <a href="product.html">product information</a>.<br> 
For a screenshot of the product <a href="../images/productimage.gif">click here</a>.</p>
</body>

By activating these links, users may visit these resources. Note that the href attribute in each source anchor specifies the address of the destination anchor with a URI. Since these resources are on the same server, relative URI's are given here.
This URI <a href="product.html"> states that the linked to resource is a web page that is in the same directory (or folder) as the web page that the link is on.
This URI <a href="../images/productimage.gif"> states that this resource is an image which (relative to the directory that the web page calling it is in), is one level up in the hierarchy defined by the path and then from there one level down in the images directory.

The destination anchor of a link may be an element within an HTML document. The destination anchor must be given an anchor name and any URI addressing this anchor must include the name as its fragment identifier.

Destination anchors in HTML documents may be specified either by the a element (naming it with the name and id attributes), or by any other element (naming with the id attribute).

Eg.1 If we had a web page whose entries on top of the page link to header elements in the same document, we could use the a element to create destination anchors as follows:

 
<p><a href="#heading1">1st Heading</a><br>
<a href="#heading2">2nd Heading</a><br>
<a href="#heading3">3rd Heading</a></p>
     
         ...the rest of the document...
<h3><a name="heading1">1st Heading</a></h3>
         ...heading1 data here...
<h3><a name="heading2">2nd Heading</a></h3>
         ...heading2 data here...
<h3><a name="heading3">3rd Heading</a></h3>
         ...heading3 data here...

Eg.2 The same effect can be achieved by making the header elements themselves the anchors by using the id attribute instead of the name attribute:

 
<p><a href="#heading1">1st Heading</a><br>
<a href="#heading2">2nd Heading</a><br>
<a href="#heading3">3rd Heading</a></p>
     
         ...the rest of the document...
<h3><id="heading1">1st Heading</a></h3>
         ...heading1 data here...
<h3><id="heading2">2nd Heading</a></h3>
         ...heading2 data here...
<h3><id="heading3">3rd Heading</a></h3>

         ...heading3 data here...

  
     
  8.3 Other link relationships  
  The most common use of a link is to retrieve another Web resource. However, one can insert links in their documents that express other relationships between resources than simply activate a link to visit that particular resource. Links that express other types of relationships have one or more link types specified in their source anchor.

The roles of a link defined by a or link are specified via the rel and rev attributes.

For instance, links defined by the link element may describe the position of a document within a series of documents. In the following example, links within the document entitled "Article 2" point to the previous and next articles:

 
<head>
<title>Article 2</title>
<link rel="prev" href="article1.html">
<link rel="next" href="article3.html">
</head>

The link type of the first link is "prev" and that of the second is "next". These are two of several recognized link types. Links specified by the link element are not rendered with the document's contents, although user agents may render them in other ways, for example, as navigation tools.

  
     
  8.4 Assigning links and anchors  
  Several HTML elements and attributes create links to other resources, for example, the img element, the form element, the applet element, the object element etc. However, this chapter discusses links and anchors created using the link and a elements. The link element may only appear in the head of a document. The a element may only appear in the body.

When the a element's href attribute is set, the element defines a source anchor for a link that may be activated by the user to retrieve a Web resource. The source anchor is the location of the a instance and the destination anchor is the Web resource.

When the name and id attributes of the a element are set, the element defines an anchor that may be the destination of other links.

The name, id and href attributes may be set simultaneously in the same a element instance.

The link element defines a relationship between the current document and another resource. The link element has no content, but the relationships it defines may be rendered by some user agents.

  
     
  8.5 Link title attribute  
  The title attribute may be set for both a and link to add information about the nature of a link. This information may be rendered as a tooltip, spoken by a user agent, etc.

A title can be supplied for each link by using the title attribute as follows:

 
<body>
<p>For more product information go to <a href="product.html" title="Click to 
go to product information page">product information</a>.<br> 
For a screenshot of the product <a href="../images/productimage.gif" title="Click for 
screenshot of product">click here</a>.</p>
</body>

Note: All the code between the opening and closing a tags should be on the same line. (The lines are broken here to prevent you from having to scroll this page horizontally while reading it).

The links are rendered with tooltips as follows (place your cursor over the links to see the tooltips):

For more product information go to product information.
For a screenshot of the product click here.

  
     
  8.6 Internationalizing links  
  Links may point to documents encoded with different character encodings, therefore the a and link elements support the charset attribute. This attribute allows you to advise user agents about the encoding of data at the other end of the link.

The hreflang attribute provides user agents with information about the language of a resource at the end of a link, just as the lang attribute provides information about the language of an element's content or attribute values.

Using this additional knowledge, user agents should be able to avoid presenting nonsense to the user. Instead, they may either locate resources necessary for the correct presentation of the document or, if they cannot locate the resources, they should at least warn the user that the document will be unreadable and explain the cause.

 

  
     
  8.7 The a element  
 

Syntax    <a>...</a>

List of a Attributes

Each a element and its attributes define an anchor as follows:

  1. The a element's content defines the position of the anchor.
  2. The name and id attributes name the anchor so that it may be the destination of other links.
  3. The href attribute makes this anchor the source anchor of exactly one link.

In the following example, the a element defines a link. The source anchor is the page designated as the "Homepage" and the destination anchor is "http://www.somesite.com/":

 
For more information please visit our <a href="http://www.somesite.com/">Homepage</a>.

This is rendered as follows:

For more information please visit our Homepage.

When a user activates this link, the user agent will retrieve the resource, which in this case, is an HTML document. The above link will place the retrieved page in the same window that it is on.

To place a linked resource (say, from an external site) into a newly opened window while keeping the original window open, add the code target="_blank" after the href value as follows:

 
For more news go to <a href="http://www.othersite.com/" target="_blank">Site's Name</a>.

Now when a user clicks on the link, a new window will open with the retrieved web page in it as illustrated below:

For more news go to Site's Name.

User agents generally render links in a way that makes them obvious to users, such as by underlining the links, etc. and the rendering may also vary according to whether the user has already visited the link or not.

We can define an anchor named "games" in the file "products.html" as follows:

 
...text preceding anchor...
<a name="games" id="games">This is the location of the anchor named games.</a>
...text after anchor...

After defining the anchor, it can be linked to from the same or another document. URIs that designate anchors contain a "#" character followed by the anchor name, known as the fragment identifier. Here are some examples of such URIs:

  • An absolute URI: http://www.somesite.com/products.html#games
  • A relative URI: ../products.html#games or just products.html#games
  • Defining the link in the same document is as follows: #games

Therefore, a link defined in the file "specifications.html" in the same directory as "products.html" would refer to the anchor as follows:

 
...text preceding anchor...
For more product information, click <a href="../products.html#games">here</a>.
...text after anchor...

The a element in the following example specifies a link (with href) and creates a named anchor (with the name and id attributes) simultaneously:

 
For a picture of the games product click
<a name="photo_games" id="photo_games" href="http://www.mysite.com/images/games.jpg">here</a>.

This example contains a link to a JPG image of the product. Activating the link will cause the image to be retrieved from the Web.

 

  
     
  8.8 Syntax used in anchor names  
  An anchor name is the value of either the name or id attribute when used in the context of anchors. Anchor names must adhere the following rules:
  • Uniqueness: Anchor names must be unique within a document. Anchor names that differ only in case may not appear in the same document.
  • String matching: Comparisons between fragment identifiers and anchor names must be done by exact (case-sensitive) match.

Therefore, the following example is correct with respect to string matching and must be considered a match by user agents:

 
<a href="#abc">anchor 1</a>
...more document text...
<a name="abc">anchor 1</a>

The following example is illegal with respect to uniqueness since the two names are the same except for case:

 
<a name="xxx">anchor 1</a>
<a name="XXX">anchor 2</a>

Although the following example is legal HTML, the behavior of the user agent is not defined. Some user agents may incorrectly consider this a match and others may not.

 
<a href="#xxx">anchor 1</a>
...more document text...
<a name="XXX">anchor 1</a>

Hence, always keep the same case (i.e. identical values) for the same anchor names and restrict them to ASCII characters.

 

  
     
  8.9 Anchors using the id attribute  
  The id attribute may be used to create an anchor at the start tag of any element including the a element.

This example illustrates the use of the id attribute to position an anchor in an h2 element. The anchor is linked to via the a element.

 
More information about this in <a href="#paragraph4">Paragraph Four</a>.
...further down in the document...
<h2 id="paragraph4">Paragraph Four</h2>
...further down in the document...
Please refer to <a href="#paragraph4">Paragraph Four</a> above for more details.

The id and name attributes share the same name space. This means that they cannot both define an anchor with the same name in the same document. However, it is permissible to use both attributes to specify an element's unique identifier for the following elements: a, applet, form, frame, iframe, img, and map. When both attributes are used on a single element, their values must be identical.

Since many existing HTML clients (user agents) don't support the use of id-type attributes in this way and the name attribute has been deprecated, identical values should be supplied for both of these attributes to ensure maximum forward and backward compatibility as follows:

<a id="foo" name="foo">...anchor description here...</a>.

The id attribute value must be unique throughout the document.

Therefore, the following example is illegal HTML since these attributes declare the same name twice in the same document.

 
<a href="#anchor1">Anchor One</a>
...further down the document...
<h3 id="anchor1">Anchor One</h3>
...further down the document...
<a name="anchor1">Anchor One</a>

Because of its specification in the HTML DTD, the name attribute may contain character entity references. The id attribute, on the other hand, may not contain character entity references.

Some points to remember when deciding whether to use the name or id attribute:

  • The id attribute can act as more than just an anchor name (e.g., style sheet selector, processing identifier, etc.).
  • The name attribute allows richer anchor names (with character entity references).

Some older user agents don't support anchors created with the id attribute. (Therefore, as already mentioned above, to ensure maximum forward and backward compatibility it is best to use both the id and name attributes, e.g.,
 <a id="foo" name="foo">...anchor description here...</a>).

 

  
     
  8.10 The base element  
 

Syntax    <base>

List of base Attributes

Links and references to external resources, e.g. images, applets, style sheets, etc. are always specified by a URI. Relative URIs are resolved according to a base URI. The base element allows you to specify a document's base URI explicitly.

When present, the base element must appear in the head section of an HTML document, before any element that refers to an external source. The path information specified by the base element only affects URIs in the document where the element appears.

For example, given the following base and a declarations:

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
<html>
<head>
<title>Products page</title>
<base href="http://www.mysite.com/products.html">
</head>
 
<body>
<p>To see our landscape paintings click <a href="../images/landscapes.gif">here</a>.<p>
</body>
</html>

the relative URI "../images/landscapes.gif" would resolve to:

http://www.mysite.com/images/landscapes.gif

Note that the object and applet elements define attributes that take precedence over the value set by the base element.
