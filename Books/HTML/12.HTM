  13. Style Sheets  
  13.1 Introduction  
  You can use HTML to markup a document's structure and style sheets to declare the document's presentation. Using style sheets solves the problem of the limited range of presentation mechanisms provided by HTML. Style sheets simplify HTML markup and allow greater influence over presentation. They give you control over the presentation of documents, e.g. font information, alignment, colors, etc. Style information can be specified for individual elements or groups of elements. It allows for the separation of content and presentation so that you can change the presentation of an entire site by editing a single style sheet, making site maintenance much easier.

Eg.1 The following example is a short style sheet stored in a file named "mystyle.css" which sets the text color of a paragraph to blue and surrounds it with a solid yellow border:

 
p.classname {
color : "blue";
border: "solid yellow";
}

You may link this style sheet to each of your HTML documents with the link element as follows:

 
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
<html>
  <head>
  <title>Document With Linked Style Sheet</title>
    <link rel="stylesheet" type="text/css" href="mystyle.css">
  </head>
  <body>
    <p class="classname">This paragraph will have blue text within a yellow border.</p>
  </body>
</html>

  13.2 Inline Style Information  
  style = property: value
This attribute specifies style information for the current element.

In CSS (cascading style sheets), property declarations have the form "property: value" and different property/value pairs are separated from each other by a semi-colon.

Eg.2 The CSS example below sets color, font family and font size information for the text in a specific paragraph.

 
<p style="font-family: arial; font-size: 10pt; color: blue">
This is how to set a particular style within a paragraph.</p>

The text is rendered as follows:

This is how to set a particular style within a paragraph.

Note: The style element (see section 9.4 below) should be used to specify style information for more than one element. For maximum flexibility it is best to define styles in external style sheets.

  
     
  13.3 The style Element

The style element allows you to put style sheet rules in the head of the document. HTML permits any number of style elements in the head section of a document.

Eg.3 In the following example, CSS style declaration puts a border around every p element in the document and centers it on the page.

 
<head>
<style type="text/css">
p {
border-width: "1"; 
border: "double"; 
border-color: "blue"; 
text-align: "center"
}
</style>
</head>

The text is surrounded by a border as follows:

The text in this paragraph is centered and surrounded by a blue double-line border.

Eg.4 We can modify it so that this style information will apply only to p elements of a specific class as follows:

 
<head>
<style type="text/css">
 p.classname {
 border-width: "1"; 
 border: "double"; 
 border-color: "blue"; 
 text-align: "center"
 }
 </style>
 </head>
 <body>
 <p class="classname">This paragraph is affected by the style</p>
 <p>This one is not affected by the style</p>
</body>

Eg.5 Finally, we can set the id attribute to limit the scope of the style information to a single instance of p as follows:

 
<head>
<style type="text/css">
 #idname {
 border-width: "1"; 
 border: "double"; 
 border-color: "blue"; 
 text-align: "center"
 }
 </style>
 </head>
 <body>
 <p class="classname">This paragraph is not affected by the style</p>
 <p id="idname">This one is affected by the style</p>
 <p>This paragraph is not affected by the style</p>
</body>

Style information may be set for almost every HTML element. However, the div and span elements are particularly useful in that they do not impose any presentation semantics (besides block-level vs. inline). When these two elements are combined with style sheets, they allow users to extend HTML indefinitely, particularly when used with the class and id attributes.

Eg.6 In the following example, we use the span element to highlight, in yellow, some words in a paragraph.

 
<head>
 <style type="text/css">
  span.hilite { background-color: "yellow" }
 </style>
</head>
<body>
  <p>The span element is used to <span class="hilite">highlight these words</span>.
</body>

The paragraph is rendered as follows:

The span element is used to highlight these words.

The same effect could be achieved using the following code:

<p>The span element is used to <span style="background-color: yellow">highlight these words</span>.</p>

Eg.7 In the following example, use is made of the div element and the class attribute to set the text alignment for a series of headings. This style information could be reused for other headings by setting the class attribute elsewhere in the document.

 
<head>
 <style type="text/css">
   div.centeralign { text-align: "center" }
 </style>
</head>
<body>
 <div class="centeralign">
   <h3>This is heading 1</h3>
 </div>
</body>

The headings are rendered as follows:

  13.4 Media types  
  You are able to design documents that take advantage of the characteristics of the media where the document is to be rendered (e.g., graphical displays, handheld devices, speech-based browsers, braille-based tactile devices, television screens, etc.). By specifying the media attribute, user agents are able to load and apply style sheets selectively.

The following is a list of recognized media descriptors:

  • screen: Intended primarily for color computer screens.
  • tty: Intended for media using a fixed-pitch character grid, such as teletypes, terminals, or portable devices with limited display capabilities. You should not use pixel units with the "tty" media type.
  • tv: Intended for television-type devices (low resolution, color, limited scrollability screens, sound available).
  • projection: Intended for projected presentations, for example projectors or print to transparencies.
  • handheld: Intended for handheld devices (typically small screen, monochrome, limited bandwidth).
  • print: Intended for paged, opaque material and for documents viewed on screen in print preview mode.
  • braille: Intended for braille tactile feedback devices.
  • embossed: Intended for paged braille printers.
  • aural: Intended for speech synthesizers.
  • all: Suitable for all devices.

The following sample declarations apply to p elements. When projected, as in a business meeting for example, all instances will have a text-alignment of left. When printed, all instances will have a text-alignment of justify.

 
<head>
 <style type="text/css" media="projection">
    p { text-align: "left"}
 </style>
 
 <style type="text/css" media="print">
   p { text-align: "justify" }
 </style>
 
 This example adds sound effects to anchors for use in speech output:
 
 <style type="text/css" media="aural">
   a { cue-before: uri(sound1.wav); cue-after: uri(sound2.wav)}
 </style>
</head>

Media control is particularly useful when applied to external style sheets since user agents can save time by downloading only those style sheets that apply to the current device. For instance, speech-based browsers can avoid downloading style sheets designed for visual browsers.

  
     
  13.5 External Style Sheets  
  It's possible to separate style sheets from HTML documents and save them as external style sheets. This offers the following benefits:
  • You may change the presentation of several HTML documents without requiring modifications to those HTML documents.

User agents may load style sheets selectively based on media descriptions.

  
     
  13.6 Specifying External Style Sheets  
  External style sheets are specified with the following attributes of the link element:
  • Set the rel attribute to "stylesheet".
  • Set the value of the type attribute to indicate the language of the linked style sheet resource. This allows the user agent to avoid downloading a style sheet for an unsupported style sheet language.
  • Set the value of href to the location of the style sheet file. The value of href is a URI.

The following specifies a link to an external style sheet located in the file mystyle.css:

<link rel="stylesheet" type="text/css" href="mystyle.css">

  13.7 Inheritance and Cascading  
  In order to render a document, the user agent needs to find values for style properties, for example the font family, font style, size, text color, etc. The exact mechanism depends on the style sheet language, but the following description is generally applicable.

The cascading mechanism is used when a number of style rules all apply directly to an element. The mechanism allows the user agent to sort the rules by specificity, to determine which rule to apply. If no rule can be found, the next step depends on whether the style property can be inherited or not. Not all properties can be inherited. For these properties the style sheet language provides default values for use when there are no explicit rules for a particular element.

If the property can be inherited, the user agent examines the immediately enclosing element to see if a rule applies to that. This process continues until an applicable rule is found. This mechanism allows style sheets to be specified compactly. For instance, you may specify the font family for all elements within the body by a single rule that applies to the body element.

 

  
     
  13.8 Precedence  
  In some cases different style sheet specifications may conflict with one another. The browser must then decide which declaration to use. The choice is made on the basis of an order of precedence.

The order of precedence for cascading style sheets is as follows:

  1. inline style information takes precedence over embedded style sheets, and
  2. embedded style sheets in turn take precedence over linked style sheets.

It is important to remember this order so that you'll know which style declarations will have precedence.
