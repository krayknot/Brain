  5. Formatting Text  
  5.1 Elements Used To Structure Text  
  Elements used to structure text are known as phrase elements and include the following:
em, strong, dfn, code, samp, kbd, var, cite, abbr, and acronym.

em: Indicates emphasis. The text is usually italicized. List of em Attributes

strong: Indicates stronger emphasis. The text is rendered in bold type. List of strong Attributes

cite: Contains a citation or a reference to other sources. List of cite Attributes

dfn: Indicates that this is the defining instance of the enclosed term. List of dfn Attributes

code: Designates a fragment of computer code. List of code Attributes

samp: Designates sample output from programs, scripts, etc. List of samp Attributes

kbd: Indicates text to be entered by the user. List of kbd Attributes

var: Indicates an instance of a variable or program argument. List of var Attributes

abbr: Indicates an abbreviated form (e.g., WWW, HTTP, URI, Mass., etc.).

acronym: Indicates an acronym (e.g., WAC, radar, etc.).

The presentation of phrase elements depends on the user agent. Generally, visual user agents present em text in italics and strong text in bold type. Speech synthesizer user agents may change the synthesis parameters, such as volume, pitch and rate accordingly.

The abbr and acronym elements allow authors to clearly indicate occurrences of abbreviations and acronyms.

  
     
  5.2 Quotations: The blockquote and q Elements  
 

Syntax    <blockquote>...</blockquote>

List of blockquote Attributes

Syntax    <q>...</q>

List of q Attributes

These two elements designate quoted text. The blockquote element is for long quotations (block-level content) and q is intended for short quotations (inline content) that don't require paragraph breaks.

The blockquote element causes the left and right margins to be pulled in for text between the opening and closing blockquote tags. The following is an example of a blockquote:

<blockquote>
<p>It was the best of times, it was the worst of times,
it was the age of wisdom, it was the age of foolishness,
it was the epoch of belief, it was the epoch of incredulity,
it was the season of Light, it was the season of Darkness,
it was the spring of hope, it was the winter of despair,
we had everything before us, we had nothing before us,
we were all going direct to Heaven, we were all going direct
the other way--in short, the period was so far like the present
period, that some of its noisiest authorities insisted on its
being received.</p>
</blockquote>

The text is rendered with the left and right margins pulled in as follows:

It was the best of times, it was the worst of times, it was the age of wisdom, it was the age of foolishness, it was the epoch of belief, it was the epoch of incredulity, it was the season of Light, it was the season of Darkness, it was the spring of hope, it was the winter of despair, we had everything before us, we had nothing before us, we were all going direct to Heaven, we were all going direct the other way--in short, the period was so far like the present period, that some of its noisiest authorities insisted on its being received.

The use of the blockquote element to indent text is deprecated in favor of style sheets. One way to do this using the <p> tag would be as follows:

<p style="margin: 0px 40px">It was the best of times, it was the worst of times,

it was the age of wisdom, it was the age of foolishness,

it was the epoch of belief, it was the epoch of incredulity,

it was the season of Light, it was the season of Darkness,

it was the spring of hope, it was the winter of despair,

we had everything before us, we had nothing before us,

we were all going direct to Heaven, we were all going direct

the other way--in short, the period was so far like the present

period, that some of its noisiest authorities insisted on its

being received.</p>

 

The text would then be rendered as follows:

It was the best of times, it was the worst of times, it was the age of wisdom, it was the age of foolishness, it was the epoch of belief, it was the epoch of incredulity, it was the season of Light, it was the season of Darkness, it was the spring of hope, it was the winter of despair, we had everything before us, we had nothing before us, we were all going direct to Heaven, we were all going direct the other way--in short, the period was so far like the present period, that some of its noisiest authorities insisted on its being received.

As you can see, it's rendered in exactly the same way as when the blockquote tags were used in the previous example.

The following example shows how the quotations are nested using the q element:

Mark said, <q>I saw Peter at work today and he said that <q>there is an

excellent movie playing at the local cinema this week.</q> I think we should go and

see it this weekend.</q>

 

Visual user agents should render it with single quotation marks around the inner quotation and double quotations marks around the outer quotation as in the example below.

Mark said, "I saw Peter at work today and he said that 'there is an
excellent movie playing at the local cinema this week.' I think we should go and
see it this weekend."

However most current browsers do not support the q element. Therefore, at present, it would be better to insert quotation marks rather than use the q element.

  5.3 The sub and sup Elements  
 

Syntax    <sub>...</sub>

List of sub Attributes

Syntax    <sup>...</sup>

List of sup Attributes

Many scientific and language scripts require subscripts (use sub element) or superscripts (use sup element) for correct rendering. The sub and sup elements should be used to markup text in these cases. For example:

Oxygen = O2  The code is O<sub>2</sub>
Water = H2O  The code is H<sub>2</sub>O
100 = 102  The code is 10<sup>2</sup>
1000 = 103  The code is 10<sup>3</sup>

  5.4 Paragraphs: The p Element  
 

Syntax    <p>...</p>

List of p Attributes

The p element represents a paragraph. It cannot contain block-level elements such as the div element and neither can it include p itself. The p element can be used as follows:

<p>"If you knew what a conflict goes on in the business mind, when the
business mind is divided between good-natured impulse and business
appearances, you would be amused, Mr. Darnay."</p>
<p>"I know, I know," rejoined Mr. Carton, carelessly.  "Don't be
nettled, Mr. Lorry.  You are as good as another, I have no doubt:
better, I dare say."</p>
<p>"And indeed, sir," pursued Mr. Lorry, not minding him, "I really
don't know what you have to do with the matter.  If you'll excuse me,
as very much your elder, for saying so, I really don't know that it is
your business."</p>

It is rendered as follows:

"If you knew what a conflict goes on in the business mind, when the business mind is divided between
good-natured impulse and business appearances, you would be amused, Mr. Darnay."

"I know, I know," rejoined Mr. Carton, carelessly. "Don't be nettled, Mr. Lorry. You are as good as another, I have no doubt: better, I dare say."

"And indeed, sir," pursued Mr. Lorry, not minding him, "I really don't know what you have to do with the matter. If you'll excuse me, as very much your elder, for saying so, I really don't know that it is your business."

  
     
  5.5 Forcing a line break: the br element  
 

Syntax    <br>

List of br Attributes

The br element forcibly breaks (ends) the current line of text. As an example:

First line<br>New second line<br>New third line<br>New fourth line

The text is rendered as follows:

First line
New second line
New third line
New fourth line

Prohibiting a line break

Sometimes you may want to prevent a line break from occurring between two words. The &nbsp; entity acts as a space where user agents should not cause a line break.

  5.6 Hyphenation  
  There are two types of hyphens in HTML: the plain hyphen and the soft hyphen. The plain hyphen should be interpreted by a user agent as a character. The soft hyphen indicates to the user agent where a line break can occur.

The plain hyphen is represented by the "-" character (or the numeric character references: &#45; or &#x2D;). The soft hyphen is represented by the named entity &shy; (or the numeric character references: &#173; or &#xAD;).

  
     
  5.7 Preformatted text: The pre element  
 

Syntax    <pre>...</pre>

List of pre Attributes

The pre element tells visual user agents that the enclosed text is "preformatted". When handling preformatted text, visual user agents:

  • Should not collapse any extra whitespaces.
  • May render text with a fixed-pitch font.
  • May disable automatic word wrap.
  • Must not disable bidirectional processing.

The following example shows text that has been preformatted:

 
<pre>
This sentence is flush left.
     This sentence is indented towards the right.
              This sentence is slightly more indented towards the right.
This sentence 
             is typed on 
                        three different lines.
</pre>

This is how the above is rendered:

This sentence is flush left.
     This sentence is indented towards the right.
              This sentence is slightly more indented towards the right.
This sentence 
             is typed on

                        three different lines.
